# Tanant Manager

## INTRO :
### An Easy interactive tool to visualize building's units.
#### in **SHORT** it works as a users-interface to channel the services that an owner can provide, to a cusomer looking for appartment to rent.
...yes, this is it, nothing fancy, yet totally helpful and effecient.

## How it works!?
### it **starts** with a *Building Owner*
> building owner draws his building and provid info..
- info such as..
1. name.
2. address.
3. floors number.
>> each floor has it's own info [floor info].
4. each appartment has it's own info [unit info].
**PS :** each part is explained below, where the upper links (floor or unit)info point at.

### the *info* is saved to the database, accessable based on user's *PERMISSION*.
- user's type are separated based on their given persmission.
1. addministrators, who are the owner of the application, the database or any **entity** that was given such permission to keep the system running smoothly.
2. owner, as in *building owner* who can construct his/her **own** buildings data for other users to **view** [view building]. **Note :** that an owner could be an administrator if he own the application's rights (mainly in cases like local use).
3. customers AKA the to become *tenant*, people who are looking for an available units to rent. so they look the does their search inside the application via *owner* (if locally) or on the net if it's a global service.
